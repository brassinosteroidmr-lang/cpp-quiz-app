## ユニット1：原材料調達（所要時間：約8分）

### 導入：なぜ重要？
原材料は「市況性」という他の調達品にはない特徴がある。価格をバイヤーがコントロールできないため、**リスク管理の知識**が問われる。試験では取引所名や価格変動対策が頻出。

### コア知識

#### 1-1 原料と材料の違い ★★

| 区分 | 定義 | 具体例 |
|------|------|--------|
| **原料** | 原産地から採取し、**一次加工**したもの | 原油、鉄鉱石、農産品 |
| **材料** | 原料を**二次加工**し、汎用用途に適合させたもの | インゴット、ビレット、鋼板、ペレット |

🔑 覚え方：「**原（げん）は一（いち）、材（ざい）は二（に）**」→ 原料＝一次加工、材料＝二次加工

#### 1-2 原材料調達の4つの関心事 ★★★

| 関心事 | ポイント |
|--------|----------|
| **市況性** | 購入価格に変動性がある。バイヤーは価格コントロールできない |
| **市場を通した取引** | 各地に取引所あり。取引価格は経済指標として扱われる |
| **採掘地・採取地の偏在** | カントリーリスク・地政学リスクの影響を受ける |
| **サステナビリティ対応** | 環境破壊、人権問題（強制労働等）への配慮が必要 |

🔑 覚え方：「**し・し・さい・サス**」（市況・市場・採掘偏在・サステナビリティ）

#### 1-3 主要取引所 ★★★

| 取引所 | 略称 | 所在地 | 主要品目 |
|--------|------|--------|----------|
| ニューヨーク商業取引所 | **NYMEX** | ニューヨーク | **原油** |
| ロンドン金属取引所 | **LME** | ロンドン | **貴金属・非鉄金属** |
| シカゴマーカンタイル取引所 | **CME** | シカゴ | **畜産物** |
| 東京商品取引所 | ― | 東京 | 貴金属・石油・農産物 |

🔑 覚え方：「**ニューヨークで油（NYMEX=原油）、ロンドンで金属（LME）、シカゴで肉（CME=畜産）**」

#### 1-4 市況に影響を与える主な要因 ★★
- 需要地域の景気動向
- 当該原材料を使った製品の販売動向
- 供給状況（不足・過剰）
- 産出国の政策・政情
- 生産に関する技術動向
- 代替品の登場
- 輸送コスト
- **外国為替動向**（日本は原料を海外依存 → **二重の変動要素**）

#### 1-5 スポット価格と長期契約価格 ★★

| 種類 | 内容 |
|------|------|
| **スポット価格** | 一回ごとの取引で成立する市場価格。余剰品の価格設定の場合も |
| **長期契約価格** | 供給者と大口需要者の間で、ある期間にわたって適用される価格 |

#### 1-6 現物市場と先物市場 ★★

| 種類 | 内容 |
|------|------|
| **現物市場** | 現実にある原料を取引する市場の価格 |
| **先物市場** | 将来の一定時期に受け渡す条件で売買契約する取引。**価格変動リスク回避策**として活用 |

#### 1-7 価格変動リスクの回避策 ★★★
1. **先物の適用** → 事前に将来の取引価格を決定
2. **長期契約** → 一定期間、価格を固定
3. **利用局面における対応**：
   - 使用量を減らす・使用をやめる
   - 材料を変える・代替品に切り替える
   - 新材料を開発する
   - 工法を変える・機構を変える

🔑 覚え方：「**先（さき）・長（なが）・利（り）**で価格変動回避」→ 先物・長期契約・利用局面対応

#### 1-8 品質の決定方法 ★
- 見本による方法
- 銘柄による方法
- 規格による方法
- 標準品による方法
- 見本と説明による方法

#### 1-9 サステナビリティ対応 ★★

| 分野 | 内容 |
|------|------|
| **環境** | 環境破壊、森林伐採、水資源過剰使用 → **環境デューデリジェンス** |
| **人権** | 紛争鉱物、児童労働、強制労働 → **人権デューデリジェンス** |
| **フェアトレード** | 発展途上国の原材料を**適正価格で継続的に調達** → 生産者・労働者の生活改善 |
| **リサイクル** | **都市鉱山** ＝ 工業製品から資源を取り出し再利用。偏在の問題がない |

### 暗記ポイント
- 取引所3つ（NYMEX/LME/CME）と品目の対応
- 価格変動リスク回避策3つ
- サステナビリティの4分野

### 確認クイズ
**Q1.** ロンドン金属取引所（LME）で主に取引される品目は何か？
→ **貴金属・非鉄金属**

**Q2.** 原材料の価格変動リスク回避策を3つ挙げよ。
→ **①先物の適用、②長期契約、③利用局面における対応**

**Q3.** 工業製品から資源を取り出し再利用することを何と呼ぶか？
→ **都市鉱山**

## ユニット2：設備調達（所要時間：約10分）

### 導入：なぜ重要？
設備調達は高額かつ全社経営に影響する。**ライフサイクルコスト**、**入手形態の比較**、**12ステップの手順**が試験で問われやすい。

### コア知識

#### 2-1 設備調達とは ★
- **設備** ＝ 機械、装置、運搬設備、試験装置・測定機器、金型、治工具、制御機器など
- **設備調達** ＝ 設備自体＋設計製作〜試験〜据付〜試運転の調達

#### 2-2 設備調達の特徴 ★★
- 全社の経営に影響 → 全関係部門の意見が必要
- 調達の関与プロセスが不明確なケースが多い
- サプライヤーが固定化しがち、随意契約も多い
- 繰返し性が低く、データ・ノウハウ蓄積がしにくい

#### 2-3 ライフサイクルコスト ★★★

| コスト区分 | 内容 | 例 |
|------------|------|-----|
| **イニシャルコスト** | 使用開始までにかかる費用 | 設備本体、据付費用 |
| **ランニングコスト** | 使用開始後の費用 | 燃料費、用力費、メンテナンス費 |
| **その他費用** | ― | 環境負荷低減費用、廃棄費用 |

🔑 覚え方：「**イ・ラ・そ**のライフサイクル」→ イニシャル・ランニング・その他

#### 2-4 入手形態の比較 ★★★

| 項目 | 買取（購入） | リース | レンタル |
|------|------------|--------|----------|
| 支払 | 一括or割賦 | 割賦払い | 割賦払い |
| 使用期間 | 長期 | 比較的長期 | **短期** |
| 対象 | バイヤー指定品 | バイヤー指定品を**リース会社が購入** | **レンタル会社の在庫**から選定 |
| 所有権 | バイヤー企業 | **リース会社** | **レンタル会社** |
| 中途解約 | ― | **原則不可** | **可能** |
| 契約終了 | ― | 返却・再リース・買取 | 返却or再レンタル |

🔑 覚え方：
- リース ＝ 「リー（理）スの通り解約不可」→ **中途解約原則不可**
- レンタル ＝ 「レン（連）絡すれば解約OK」→ **中途解約可能**

#### 2-5 サプライヤー・価格決定方法 ★★

| 方法 | 内容 |
|------|------|
| **随意契約**（一社特命） | 特例措置。原則は複数見積 |
| **指名競争入札** | 複数社を指名、最低価格提示者と契約 |
| **相見積契約** | 複数の全見積先と価格交渉し契約 |
| **交渉優先権契約** | 最低価格提示の数社と交渉 → **CR効果が大きい**（時間はかかる） |

#### 2-6 設備調達の12ステップ ★★★

| No. | ステップ | ポイント |
|-----|----------|----------|
| ① | **仕様書作成** | ライフサイクルコスト最小化、VE的アプローチ確保 |
| ② | **サプライヤー候補選定** | 過去実績に縛られず広く門戸開放 |
| ③ | **設備仕様説明会** | 原則3社以上、公正・公平な運営 |
| ④ | **提案説明会** | サプライヤー候補が個別に提案 |
| ⑤ | **見積書取得** | 項目統一・細分化、差替え再提出禁止 |
| ⑥ | **サプライヤー候補絞込み** | 2〜3社に絞込み |
| ⑦ | **コスト分析** | 各社見積項目別比較、類似品比較 |
| ⑧ | **調達交渉** | ― |
| ⑨ | **契約** | 基本契約書＋高額は個別契約書 |
| ⑩ | **納期管理** | 工程表提出、完成図書管理 |
| ⑪ | **初期流動管理** | 受注者とともに実施 |
| ⑫ | **固定資産管理** | 20万円以上は固定資産計上 |

🔑 覚え方：「**し・さ・せ・て・み・さ・コ・ちょう・けい・の・しょ・こ**」
→ 仕様書・サプライヤー・説明会・提案・見積・絞込み・コスト・調達交渉・契約・納期・初期流動・固定資産

#### 2-7 設備仕様書の3種類 ★★
1. **要求仕様書**：発注者 → 受注者に仕様提示
2. **提案仕様書**：受注者 → 発注者に仕様提案
3. **確定仕様書**：双方協議のうえ発注者が承認

#### 2-8 設備能力を表す指標 ★★

| 指標 | 内容 |
|------|------|
| **タクトタイム** | 機械スピード能力。生産能力を示す |
| **MTBF** | Mean Time Between Failures＝**平均故障間隔**。前回故障から今回故障までの実働時間総和÷故障発生件数 |
| **歩留り** | 良品数÷（投入数−他要因不良数） |
| **工程能力（Cp）** | （規格上限−規格下限）÷6σ。通常 **Cp≧1.33（4σ）** が要求 |

### 確認クイズ
**Q1.** 設備の入手形態で、中途解約が原則不可なのはどれか？
→ **リース**

**Q2.** 設備調達の手順において、原則3社以上のサプライヤーを選定するのはどのステップか？
→ **③設備仕様説明会**

**Q3.** MTBFとは何の略か？日本語で何と呼ぶか？
→ **Mean Time Between Failures＝平均故障間隔**

## ユニット3：ソフトウェア調達（所要時間：約10分）

### 導入：なぜ重要？
ソフトウェアは「目に見えない」調達品。価格決定法が**8種類以上**あり、さらに**知的財産権**も問われる。暗記量が多いが試験頻出。

### コア知識

#### 3-1 ソフトウェア調達の種類 ★

| 区分 | 内容 |
|------|------|
| **パッケージソフト購入** | サプライヤー作成のソフトを購入（使用権の許諾） |
| **ライセンス導入** | 技術使用料を支払い |
| **SaaS** | 開発会社サーバーのアプリをクラウドで利用（Software as a Service） |
| **開発外部委託** | 外部企業に開発を委託（製造委託に相当） |

購入形態の変遷：光記憶媒体 → ダウンロード → **クラウドサービス（SaaS）**

#### 3-2 開発方法 ★★

| 方法 | 特徴 |
|------|------|
| **ウォーターフォール型** | 上流から下流へ順次進行。7工程を順番に実施 |
| **アジャイル型** | 機能単位で、要件定義→テストまでを**繰り返す** |

**ウォーターフォール型の7工程** ★★★

| No. | 工程 | 内容 |
|-----|------|------|
| ① | 要件定義 | ユーザー要求事項の分析、システム化事項のまとめ |
| ② | 外部設計 | 帳票・画面などユーザーインターフェイス設計 |
| ③ | 内部設計 | プログラム詳細機能、内部構造の設計 |
| ④ | プログラム設計 | コーディング用の仕様設計 |
| ⑤ | プログラミング | ソースコード作成 |
| ⑥ | 単体テスト | プログラムの動作確認 |
| ⑦ | 運用テスト | 運用マニュアルに従ったシステム利用 |

🔑 覚え方：「**よ・が・な・プ・プ・た・う**」→ 要件・外部・内部・プログラム設計・プログラミング・単体テスト・運用テスト

#### 3-3 調達価格の決定法 ★★★

| 方式 | 内容 |
|------|------|
| **単金方式** | 単金×工数で算出 |
| **類似法** | 過去の類似システム実績をもとに推定 |
| **三点見積** | **（最小値＋最尤値×4＋最大値）÷6** |
| **トップダウン見積** | 全体規模→サブシステムに割り振り |
| **ボトムアップ見積** | サブシステムごとに推定→積み上げ |
| **プログラムステップ法** | プログラムの行数で開発費を見積もる |
| **開発生産性法** | 開発費＝単金×（規模÷単位時間当り生産性） |
| **ファンクションポイント法** | 入出力・ファイル数等に重み付け→過去実績で算出 |

**モデル化法**：
- **COCOMO法**：開発規模（ステップ数）と難易度で工数・期間を見積もる
- **Dotyモデル**：ソースコードのステップに基づく指数関数モデル
- **Putnamモデル**：全体コスト→工程・サブシステム単位で見積もる

🔑 三点見積の公式は暗記必須：**(最小＋最尤×4＋最大)÷6**

#### 3-4 派遣と請負の違い ★★★

| 項目 | 派遣 | 請負 |
|------|------|------|
| 契約 | 労働者派遣契約 | 請負契約 |
| 雇用関係 | **派遣会社**と労働者 | **請負会社**と労働者 |
| 指揮命令 | **派遣先**会社が行う | **請負会社**が行う |
| 成果物責任 | なし | **あり**（仕事の結果に対して報酬） |

🔑 覚え方：「**派遣は先（さき）が指揮、請負は元（もと）が指揮**」

#### 3-5 著作権 ★★★

**著作者人格権**（譲渡・相続**不可**）：

| 権利 | 内容 |
|------|------|
| **公表権** | 発表するか否かを決める権利 |
| **氏名公表権** | 氏名を表示するか否かを決める権利 |
| **同一性保持権** | 内容を勝手に変更されない権利 |

**著作財産権**（譲渡・相続**可能**）：複製権、上演権・演奏権、翻訳権 等

🔑 覚え方：「**人格は譲れない（譲渡不可）、財産は譲れる（譲渡可）**」

#### 3-6 工業所有権 ★★★

| 権利 | 対象 | 保護期間 |
|------|------|----------|
| **特許** | 高度な発明（物・方法・生産方法の3タイプ） | **出願から20年** |
| **実用新案** | 物品の形状・構造・組合せ（発明ほど高度でない） | **出願から6年** |
| **意匠** | 物品の形状・模様・色 | **登録から15年** |
| **商標** | 商品・サービスを区別するマーク | **登録から10年**（更新可） |

🔑 覚え方：「**特（とく）に（20）実（じつ）は6（ろく）でもない、意匠（い）15（いちご）商標（しょう）10（じゅう）**」
→ 特許20年、実用新案6年、意匠15年、商標10年

注意：特許・実用新案は「**出願**から」、意匠・商標は「**登録**から」起算

### 確認クイズ
**Q1.** 三点見積の計算式を答えよ。
→ **(最小値＋最尤値×4＋最大値)÷6**

**Q2.** 派遣において、労働者に指揮命令を行うのはどの会社か？
→ **派遣先会社**

**Q3.** 工業所有権のうち、出願から6年の保護期間を持つのは何か？
→ **実用新案**

## ユニット4：プロジェクト調達（所要時間：約5分）

### 導入：なぜ重要？
プロジェクト調達は「現物それ以外に同じものがない」一品もの。建設・プラント・ITの3領域と新しい契約方式が出題ポイント。

### コア知識

#### 4-1 プロジェクト調達の3領域 ★★

| 領域 | 具体例 |
|------|--------|
| **建設** | オフィスビル、住宅、工場、施設、道路 |
| **プラント** | 石油プラント、水処理プラント、化学工場 |
| **ITシステム** | 情報システム、通信システム |

#### 4-2 建設プロジェクトの新しい契約方式 ★★★

| 方式 | 内容 |
|------|------|
| **CM方式** (Construction Management) | 発注者の代行者・補助者である**コンストラクトマネージャー**を置き、技術検討や工程管理等のマネジメント業務を行う |
| **ECI方式** (Early Contractor Involvement) | プロジェクトの**設計段階から施工者の技術力を反映** → コスト削減・納期短縮 |

🔑 覚え方：
- CM ＝ 「**C**onstruction の**M**anager（管理者）を置く」
- ECI ＝ 「**E**arly（早期に）**C**ontractor（施工者）を**I**nvolve（巻き込む）」

#### 4-3 プラントプロジェクト ★★
- **EPC** ＝ Engineering（設計）、Procurement（調達）、Construction（建設）
- エンジニアリング会社がE・P・Cを請け負う
- **フルターンキー契約** ＝ 設計から調達、建設、試運転、性能保証まで一括請負

🔑 覚え方：「**フルターンキー ＝ 鍵を回せばすぐ使える**」→ 全部おまかせ契約

#### 4-4 プロジェクト調達の核心 ★★
- **スケジュール・納期管理が最重要**
- 資機材の遅れ → 据付作業人員や重機の日程調整が発生
- 変更はつきもの → 外部要因によるスケジュール調整を想定しておく

#### 4-5 プロジェクト調達における調達物 ★
- **ハードウェア系**：資材・機材（設備・器具・重機）、間接材（事務用品、MRO系）
- **サービス系**：業務委託（物流、警備、清掃、システム開発、コンサル）
- MRO ＝ Maintenance, Repair, Operation に用いる副資材

#### 4-6 プロジェクトにおける契約形態 ★
建設プロジェクトの流れ：施主（企業・組織）→ 建設会社 → 下請（各「→」が調達行為）

### 確認クイズ
**Q1.** 設計段階から施工者の技術力を反映させる建設契約方式は何か？
→ **ECI方式（Early Contractor Involvement）**

**Q2.** EPCの「P」は何を指すか？
→ **Procurement（調達）**

**Q3.** プロジェクト調達における最も重要な管理事項は何か？
→ **スケジュール・納期管理**

## ユニット5：仕入（所要時間：約5分）

### 導入：なぜ重要？
「仕入」は小売業特有の調達。製造業の調達との**決定的な違い**と、仕入形態3種の比較が試験で問われる。

### コア知識

#### 5-1 仕入の特徴 ★★
- 調達した商品が**そのまま売り場に並ぶ** → 消費者の審判を受ける
- 仕入担当者 ＝ **バイヤー**
- **マーチャンダイザー** ＝ 商品開発計画、販売商品決定、サプライヤー折衝、取引条件決定に責任
- **目利き**と**商品知識**が重要

#### 5-2 仕入先 ★

| 仕入先 | メリット |
|--------|----------|
| **仲介業者（卸・問屋）** | 在庫保有、決済機能、数量調整、情報提供 |
| **メーカー・生産者** | 中間マージン省略 |

#### 5-3 仕入方法 ★★

| 方法 | 特徴 |
|------|------|
| **大量仕入** | 同一商品を大量に購入。価格低減メリットあるが在庫過多リスク |
| **当用仕入** | 必要の都度発注。売残りリスク低減、累積仕入量割引交渉も可能 |
| **本部集中仕入** | チェーンストア本部が全店舗分を一括購入 |
| **店舗分散仕入** | 各店舗で分散して仕入れ。地域状況を反映 |
| **集約仕入** | 仕入先を絞り1社あたりの量を増やし単価低減 |

#### 5-4 仕入形態3種 ★★★

| 形態 | 所有権 | 小売のリスク | メーカーのリスク |
|------|--------|-------------|-----------------|
| **①買取仕入** | 商品引渡で**小売へ移転** | **在庫・売残りリスクあり** | 返品リスクなし |
| **②委託** | **メーカー**にある | 売残りリスクなし | **返品リスクあり** |
| **③消化** | **メーカー**にある（保管責任も） | なし | **在庫・返品リスクあり** |

🔑 覚え方：「**買（か）い取ったら自分のもの、委（い）託・消（しょう）化はメーカーのもの**」
- 買取 → 所有権が小売に移る → 小売がリスクを負う
- 委託・消化 → 所有権はメーカー → メーカーがリスクを負う

消化仕入の特徴：**売れた分だけ仕入れたことになる**

### 確認クイズ
**Q1.** 商品の所有権がメーカーにあり、売れた分だけ仕入れたことになる形態は何か？
→ **消化仕入**

**Q2.** 商品開発計画から取引条件決定まで責任を持つ担当者を何と呼ぶか？
→ **マーチャンダイザー**

**Q3.** 買取仕入において、在庫・売残りリスクを負うのはどちらか？
→ **小売側**

## ユニット6：間接品・サービス調達（所要時間：約8分）

### 導入：なぜ重要？
間接品は「目立たないが合計すると大きな金額」になる領域。**8つの課題**と**管理力強化の4ステップ**、CR施策が出題のポイント。

### コア知識

#### 6-1 間接品・サービスの分類 ★

| 区分 | 例 |
|------|-----|
| **間接品** | 事務用品、印刷物、IT関係、販促用品、工場消耗品 |
| **サービス（利用）** | 電力・ガス・水、交通、通信、ITサービス |
| **サービス（業務委託）** | 清掃、警備、工事、物流 |

#### 6-2 間接品・サービス調達の8つの課題 ★★★

| No. | 課題 |
|-----|------|
| ① | 管理が**利用部門に任されている** |
| ② | **購入状況の把握が難しく**、発生実態が見えない |
| ③ | **利用者が多岐**にわたっている |
| ④ | **サプライヤーが固定化**している |
| ⑤ | 購入**価格にばらつき**がある |
| ⑥ | 業務上不要な、**過剰性能**の品目がある |
| ⑦ | **コンプライアンスリスク**がある |
| ⑧ | **CRノウハウ**の社内蓄積が不足している |

🔑 覚え方：「**り・こう・り・さ・か・か・コ・CR**」
→ 利用部門任せ・購入状況・利用者多岐・サプライヤー固定・価格ばらつき・過剰性能・コンプライアンス・CRノウハウ不足

#### 6-3 管理力強化の4ステップ ★★★

| Step | 内容 |
|------|------|
| **4-1** | **CR推進の組織体制確立**（集中的にCRを進める組織を決定、関与率を指標管理） |
| **4-2** | **調達状況の把握**（誰がどこから何を購入しているかのデータ整備） |
| **4-3** | **対象アイテムの絞込み**（**ABC分析**で金額の大きな品目から優先） |
| **4-4** | **施策の検討**（CR施策の立案と実行） |

🔑 覚え方：「**組（そ）・状（じょう）・絞（しぼ）・施（し）**」→ 組織・状況把握・絞込み・施策

#### 6-4 CR施策の体系 ★★

**(1) 仕様の見直し・VE**
- ①仕様適正化（過剰仕様の見直し）
- ②VE

**(2) コスト低減**
- ①CRアイデアの探索（展示会・セミナー参加）
- ②ベンチマークによるCR（他サプライヤーからの競争見積）
- ③コスト分析によるCR（要素別に推計・積み上げ）

**(3) 買い方の工夫**
- ①集約（サプライヤーを絞り→ボリュームディスカウント）
- ②購入方法の見直し（取引ルート、購入タイミング、数量）
- ③**バンドリングとアンバンドリング**
- ④リバースオークションの活用

**(4) 利用量コントロール**
- ①利用サイドの利用量節減（**「買わないことが最大のCR」**）
- **ナッジ**の活用 ＝ 「ちょっとしたきっかけを与えて行動を促す」

#### 6-5 バンドリングとアンバンドリング ★★

| 手法 | 内容 | 例 |
|------|------|-----|
| **バンドリング** | 複数品目をまとめて**1社に発注** | 広報資料の制作＋印刷を1社にまとめて発注 |
| **アンバンドリング** | 発注単位を**分離して個別発注** | 制作と印刷を分離し、それぞれのコストを精査 |

#### 6-6 購入パターン3種 ★★

| パターン | 内容 |
|----------|------|
| **都度調達品目** | 案件の都度、サプライヤー選定・価格決定 |
| **推奨サプライヤー都度調達** | 調達部門推奨のサプライヤーに見積依頼し購入 |
| **継続調達品目（カタログ調達）** | 年間契約済、ユーザーは簡易プロセスで購入 |

#### 6-7 間接品調達のIT活用 ★
間接品調達に特化したITシステムの主な機能：
1. 多様なプロセスのシンプル化
2. 集中購買の実現支援
3. 見積入手・比較の簡易化と情報蓄積
4. 逆オークションなどの価格決定支援
5. カタログ機能
6. 価格情報などの比較可能なコミュニティー形成

### 確認クイズ
**Q1.** 間接品・サービス調達の管理力強化で、ABC分析を用いるのはどのステップか？
→ **4-3 対象アイテムの絞込み**

**Q2.** 発注単位を分離して個別発注し、コストを精査する手法は何か？
→ **アンバンドリング**

**Q3.** 消耗品の在庫場所に購入価格を掲示して節約を促す手法は、行動経済学の何を活用したものか？
→ **ナッジ**

## ユニット7：業務委託（所要時間：約5分）

### 導入：なぜ重要？
業務委託はサービス調達の一種。サービスの**4つの特性**と**2つのタイプ**、管理のポイントが試験に出る。

### コア知識

#### 7-1 業務委託とは ★
- 企業が自社の業務の一部を外部に委託すること
- 「丸投げ」してはならない
- 理由：本業集中、コスト低減、外部専門性活用、人件費削減

#### 7-2 サービスの4つの特性 ★★★

| No. | 特性 | 内容 |
|-----|------|------|
| ① | **無形性** | サービスには形がない |
| ② | **同時性** | 生産と消費が同時に発生する |
| ③ | **消滅性** | 提供と同時に消滅する |
| ④ | **変動性** | 提供者・時間・場所により品質に変動がある |

🔑 覚え方：「**む・ど・しょう・へん**」→ 無形・同時・消滅・変動
または「**サービスは『むどうしょうへん』（無同消変）**」

#### 7-3 サービスの2タイプ ★★

| タイプ | 特徴 | 例 |
|--------|------|-----|
| **プロセス型** | サービス内容やプロセスが**予め定められている** | 物流、コールセンター |
| **プロフェッショナル型** | 委託元の要望に応じて提供企業が**内容を設計** | 弁護士、コンサルティング |

🔑 覚え方：「**プロセス＝手順が決まっている（定型）、プロフェッショナル＝プロの判断（非定型）**」

#### 7-4 業務委託の契約 ★★

| 契約形態 | 内容 |
|----------|------|
| **請負契約** | 委託先が**成果物を完成**させ、委託元は対価を払う |
| **準委任契約** | 法令以外の事務を委任。**成果物に対する責任は問われない** |

🔑 覚え方：「**請負は『成果で請け負う』、準委任は『過程のお任せ』**」

#### 7-5 業務委託の管理 ★★
1. **業務内容の明示**：委託する業務内容を明確に文書化（作業範囲、作業場所、実施期間、成果物、受入基準等）
2. **定期的な入札**：委託先の固定化防止。ただし**スイッチングコスト**も考慮
3. **KGI・KPIの設定**：パフォーマンスをモニターできる仕組み

### 確認クイズ
**Q1.** サービスの4つの特性を全て答えよ。
→ **①無形性、②同時性、③消滅性、④変動性**

**Q2.** 成果物に対する責任が問われない契約形態は何か？
→ **準委任契約**

**Q3.** 物流やコールセンターのように、プロセスが予め定められているサービスのタイプは何か？
→ **プロセス型**

## まとめ：専門領域調達の全体像

専門領域調達は7つの異なる調達カテゴリーを横断的に学ぶ分野。各領域の**固有の特徴**と**共通する調達の基本原則**を結びつけて理解するのがコツ。

| 領域 | 最大の特徴 | 試験最頻出ポイント |
|------|-----------|-------------------|
| 原材料 | 市況性（価格コントロール不可） | 取引所、リスク回避策、サステナビリティ |
| 設備 | 高額・ライフサイクルコスト | 入手形態比較、12ステップ、MTBF |
| ソフトウェア | 目に見えない | 価格決定法、知的財産権、派遣vs請負 |
| プロジェクト | 一品もの | CM/ECI方式、EPC、納期管理 |
| 仕入 | 消費者の審判 | 仕入形態3種の所有権・リスク |
| 間接品 | 見えにくいコスト | 8課題、4ステップ、バンドリング |
| 業務委託 | サービス特性 | 4特性、2タイプ、契約形態 |
